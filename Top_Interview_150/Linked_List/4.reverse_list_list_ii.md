
# ================================================
# REVERSE LINKED LIST II
# CLI VERSION + INTERVIEW LOGIC
# ================================================

# ------------------------------------------------
# Problem Statement
# ------------------------------------------------

Given the head of a singly linked list and two integers:

left and right

Reverse the nodes of the list from position left to position right.

Do this in-place.

Return the head of the modified list.

---

Example:

Input:

1 → 2 → 3 → 4 → 5  
left = 2  
right = 4

Output:

1 → 4 → 3 → 2 → 5

---

# ------------------------------------------------
# Key Observations
# ------------------------------------------------

• Only the sublist between left and right should be reversed.  
• Nodes before left remain unchanged.  
• Nodes after right remain unchanged.  
• We must reverse POINTERS, not values.  
• This must be done in O(1) space.

---

# ------------------------------------------------
# Important Concept
# ------------------------------------------------

Split the list into three logical segments.

A → B → C

Where:

A = nodes before left  
B = nodes between left and right  
C = nodes after right

Example:

1 → 2 → 3 → 4 → 5  
    L         R

Segments:

A = 1  
B = 2 → 3 → 4  
C = 5

Goal:

A + reverse(B) + C

Result:

1 → 4 → 3 → 2 → 5

---

# ------------------------------------------------
# Why Dummy Node is Important
# ------------------------------------------------

Dummy simplifies edge cases when:

left = 1

Without dummy:

The head pointer would change.

With dummy:

dummy → head

We always return:

dummy.next

---

# ------------------------------------------------
# Core Idea of the Algorithm
# ------------------------------------------------

We maintain three important pointers.

prev  
curr  
temp

Definitions:

prev → node before the reversing segment  
curr → first node of the segment  
temp → node that will be moved to front

Important behavior:

prev NEVER moves  
curr stays fixed  
temp keeps changing

---

# ------------------------------------------------
# Pointer Structure
# ------------------------------------------------

Initial:

prev → curr → temp → ...

Example:

1 → 2 → 3 → 4 → 5  
     prev curr temp

Goal:

Move temp directly after prev.

---

# ------------------------------------------------
# Reversal Trick (Front Insertion)
# ------------------------------------------------

Operation:

temp = curr.next  
curr.next = temp.next  
temp.next = prev.next  
prev.next = temp

Effect:

Take the next node and insert it after prev.

---

# ------------------------------------------------
# Step-by-Step Example
# ------------------------------------------------

Initial:

1 → 2 → 3 → 4 → 5  
     prev curr temp

Step 1:

Move node 3.

1 → 3 → 2 → 4 → 5

prev = 1  
curr = 2

---

Step 2:

Move node 4.

1 → 4 → 3 → 2 → 5

Segment reversed.

---

# ------------------------------------------------
# Why This Works
# ------------------------------------------------

We repeatedly move the next node to the front.

Transform:

2 → 3 → 4

Into:

4 → 3 → 2

Without breaking the list.

---

# ------------------------------------------------
# Correct Python Implementation
# ------------------------------------------------

class Solution:
    def reverseBetween(self, head, left, right):

        if not head or left == right:
            return head

        dummy = ListNode(0)
        dummy.next = head

        prev = dummy

        # Move prev to node before left
        for _ in range(left - 1):
            prev = prev.next

        curr = prev.next

        # Reverse sublist
        for _ in range(right - left):

            temp = curr.next
            curr.next = temp.next
            temp.next = prev.next
            prev.next = temp

        return dummy.next

---

# ------------------------------------------------
# Loop Invariant (Interview Explanation)
# ------------------------------------------------

At every iteration:

prev → reversed part → curr → remaining

The reversed portion grows by one node each step.

---

# ------------------------------------------------
# Common Mistakes
# ------------------------------------------------

❌ Reversing the entire list.

❌ Swapping values instead of pointers.

❌ Moving prev pointer during reversal.

❌ Forgetting dummy node.

❌ Accessing .next.next without checking.

---

# ------------------------------------------------
# Time & Space Complexity
# ------------------------------------------------

Time Complexity: O(n)

Space Complexity: O(1)

# ------------------------------------------------
# Key Takeaways
# ------------------------------------------------

• Always manipulate pointers, not values.  
• Use dummy node to simplify edge cases.  
• Keep prev fixed as anchor.  
• curr remains the start of unreversed segment.  
• temp is repeatedly inserted at front.  
• Classic in-place linked list reversal trick.
